# 常见问题速查表（Troubleshooting）

> 基于 `pay-common/README.md` §⚠️注意事项 + `开发注意事项与模板使用指南.md` §五 整理。
> 共收录 **20+** 个常见问题及解决方案。

## 一、签名相关（最高频）

### 1.1 签名失败 / 验签不通过

| # | 可能原因 | 症状表现 | 排查方式 | 解决方案 |
|---|---------|---------|---------|---------|
| 1 | **PEM 私钥格式错误** | 启动日志显示 `privateKey 包含真换行: false` | 查看 `config.js` 日志 | 确保用 `\n`（两字符）而非真换行；运行 `check_pem_format.py` 检查 |
| 2 | **公钥搞混** | 验签始终失败 | 检查 `wxPayPublicKey` 来源 | 必须用**微信支付公钥**（商户平台→API安全→微信支付公钥），不是商户 API 公钥 |
| 3 | **证书序列号错误** | 签名时 serial 不匹配 | 重新从商户平台确认序列号 | 复制完整的 40 位十六进制序列号 |
| 4 | **下单与调起用了不同私钥** | `requestPayment` 报签名错误 | 检查下单和调起签名是否用同一把 `privateKey` | 统一使用同一份私钥文件 |

### 1.2 环境变量导致的签名问题

| # | 问题 | 原因 | 解决方案 |
|---|------|------|---------|
| 5 | `privateKey` 含非法字符 | 复制粘贴引入了空格或特殊字符 | 清除首尾空格，确保只有 `\n` 分隔各行 |
| 6 | `apiV3Key` 长度不对 | 必须恰好 32 字节 | 重新从商户平台获取并确认长度 |
| 7 | 环境变量未生效 | 修改了 .env 但未重启服务 | 重启 `npm start` 或重新部署 |

## 二、回调相关（次高频）

### 2.1 收不到回调

| # | 可能原因 | 排查方式 | 解决方案 |
|---|---------|---------|---------|
| 8 | **回调 URL 不可达** | curl 返回连接超时/拒绝 | `curl -v <URL>` 测试连通性；运行 `test_callback_url.sh` | 确保 URL 为 HTTPS 公网可达；本地调试用 ngrok |
| 9 | **signMode 与 URL 不匹配** | `signMode=sdk` 但 URL 指向集成中心（或反之） | 检查 `.env` 中 `signMode` 和三个 `notifyURL*` | 统一方向：sdk→自己，gateway→集成中心 |
| 10 | **URL 带 `?` 参数** | 微信直接拒绝请求 | 检查 URL 中是否有 `?` 或 `&` | 去掉参数，使用纯净路径 |
| 11 | **HTTP 而非 HTTPS** | 微信只支持 HTTPS 回调 | 确认协议 | 云函数需开启 HTTP 访问服务获取 HTTPS 域名 |
| 12 | **云函数未开启 HTTP 访问服务** | 域名不可达 | 控制台确认 | 进入云开发控制台 → 云函数 → 开启 HTTP 访问服务 |
| 13 | **SDK 模式下回调路由开了身份认证** | 微信无 Token，被 401/403 | 检查路由配置 | 支付/转账回调路由**不能**开鉴权 |
| 14 | **IP 白名单未放行** | 防火墙拦截了微信 IP | 确认安全组/防火墙规则 | 放行微信回调 IP 段（上海/深圳/广州腾讯云） |

### 2.2 回调重复

| # | 原因 | 是否正常 | 处理建议 |
|---|------|:-------:|---------|
| 15 | **微信重试机制** | 正常 | 同一笔订单最多收到 ~15 次回调（间隔逐步拉长）；必须实现幂等逻辑 |
| 16 | **未返回 SUCCESS 应答** | 异常 | 确认回调接口 5 秒内返回 `{code:"SUCCESS"}`；模板已实现先应答后处理 |

## 三、前端集成问题

### 3.1 小程序

| # | 问题 | 原因 | 解决方案 |
|---|------|------|---------|
| 17 | **登录失败** | `ENV_ID` 未替换或小程序身份源未开启 | 检查前端 `app.js` 配置；确认控制台已开启小程序身份源 |
| 18 | **openid 缺失** | 未使用 `signInWithOpenId()` 登录 | 必须通过 `signInWithOpenId()` 获取，不能用 `wx.login()` 的 code 换取 |
| 19 | **`requestPayment` 签名错误** | 后端返回的 `paySign` 签名串有问题 | 确认下单与调起使用同一私钥；检查 `package` 字段格式（需 `prepay_id=xxx`） |
| 20 | **云 API 调用 401/403** | accessToken 过期或无效 | 检查 token 刷新机制，调用 `reLogin()` 重新登录 |
| 21 | **request 合法域名未配置** | `wx.request` 被拦截 | 小程序后台 → 开发设置 → 服务器域名 → 添加 `tcloudbasegateway.com` |

### 3.2 H5

| # | 问题 | 原因 | 解决方案 |
|---|------|------|---------|
| 22 | **H5 "当前页面无法发起支付"** | 支付授权目录未配置或不匹配 | 商户平台 → 产品中心 → 开发配置 → 支付授权目录 → 添加域名 |
| 23 | **H5 场景信息缺失** | 微信风控拦截 | 必须传 `scene_info.payer_client_ip`（用户真实 IP）+ `h5_info.type`（如 `Wap`）|

### 3.3 APP

| # | 问题 | 原因 | 解决方案 |
|---|------|------|---------|
| 24 | **APP 签名参数格式错误** | APP 与 JSAPI 签名格式不同 | APP 的 prepay_id 不带前缀、签名字段叫 `sign` 非 `paySign`、需 partnerId |

### 3.4 Native 扫码

| # | 问题 | 原因 | 解决方案 |
|---|------|------|---------|
| 25 | **二维码过期** | `code_url` 有效期约 2 小时 | 过期后重新下单；前端应有超时提示 |
| 26 | **扫码后不知道结果** | Native 没有前端回调 | 需轮询查单接口（`wxpay_query_order_by_out_trade_no`），间隔 2-3s |

## 四、部署问题

| # | 问题 | 原因 | 解决方案 |
|---|------|------|---------|
| 27 | **部署后 502** | 冷启动超时 / 环境变量缺失 / 内存不足 | 运行 `validate_env.sh` 检查环境变量；增加云函数内存；等待冷启动完成再测 |
| 28 | **tcb fn deploy 失败** | `cloudbaserc.json` 配置错误 / 未登录 | 检查 `envId` 是否正确；执行 `tcb login` |
| 29 | **环境变量未生效** | 通过命令行设的环境变量优先级低于控制台 | 推荐在控制台配置环境变量；或在 `cloudbaserc.json` 的 `envVariables` 中配置 |
| 30 | **云托管构建失败** | Dockerfile 问题 / 依赖安装失败 | 检查 `package.json` 依赖版本兼容性；查看构建日志 |

## 五、转账相关问题

| # | 问题 | 原因 | 解决方案 |
|---|------|------|---------|
| 31 | **转账受理成功但实际没到账** | `ACCEPTED` ≠ 转账成功 | 必须查单或等回调确认最终状态；用户 24h 内未确认则自动关单退款 |
| 32 | **"金额不在允许范围内"** | 单笔金额超出免密范围（≥2000 元或 <0.3 元） | 调整金额到 0.3~1999.99 元范围；或实现敏感字段加密后支持 ≥2000 元 |
| 33 | **"商户号错误"** | 前端 `requestMerchantTransfer` 的 mchId 为空或错误 | mchId 必须从后端返回数据中获取（后端透传），不要硬编码 |
| 34 | **频率超限 / SYSTEM_ERROR** | 请求过于频繁 | 用相同参数 + 相同 `out_bill_no` 重试；同一单号重试期为 3 个自然日 |

## 六、快速诊断流程

遇到问题时，按以下顺序排查：

```
Step 1: 确定问题类型
├── 签名/验签失败 → 看第一节
├── 回调不通 → 看第二节
├── 前端报错 → 看第三节
├── 部署异常 → 看第四节
└── 转账问题 → 看第五节

Step 2: 运行诊断脚本（如有）
├── env 配置问题 → bash scripts/validate_env.sh .env
├── PEM 格式问题 → python3 scripts/check_pem_format.py '<key>'
├── 回调连通性 → bash scripts/test_callback_url.sh <url>
└── 部署一致性 → python3 scripts/check_deploy_config.py <project_dir>

Step 3: 查看日志
├── 本地开发 → 终端输出
├── 云函数 → CloudBase 控制台 → 日志
└── 云托管 → CloudBase 控制台 → 云托管 → 日志

Step 4: 对照速查表定位
    → 找到对应条目 → 执行解决方案
```

## 七、收集有效信息的技巧

向用户排查问题时，请按以下模板收集信息：

| 信息项 | 为什么需要 | 获取方式 |
|--------|-----------|---------|
| **完整错误信息** | 精准定位问题 | 控制台截图/复制文本；服务端日志 |
| **signMode 值** | 区分排查方向 | 查看 `.env` 或启动日志 |
| **部署方式** | 决定排查路径 | 云函数 / 云托管 / 本地 |
| **支付方式** | JSAPI/H5/Native/APP 各有不同的坑 | 用户描述 |
| **操作步骤** | 复现问题 | 用户描述触发条件 |
| **环境变量脱敏版** | 排查配置问题（隐藏私钥等敏感信息） | `.env` 文件（删除 privateKey 等值后提供） |

*深度错误分析见 [error-patterns.md](error-patterns.md)*
