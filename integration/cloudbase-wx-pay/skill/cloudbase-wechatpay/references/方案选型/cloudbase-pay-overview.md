# CloudBase 支付方案全景与选型指南

> 基于 `pay-common-方案对比分析.md` + `微信支付三种方式对比报告.md` 整理。
> 帮助用户在 CloudBase 平台上选择最合适的微信支付接入方案。

---

## 一、CloudBase 微信支付能力总览

```
CloudBase 微信支付生态
├── 方案 A：小程序云调用（免证书）
│   ├── wx-server-sdk 私有协议
│   ├── 免商户 API 证书、免签名
│   └── ⚠️ pay-common 暂不支持（预留）
│
├── 方案 B：pay-common 模板（推荐）⭐
│   ├── JSAPI / H5 / Native / APP 四合一
│   ├── HTTP 云函数 / 云托管 / 本地 三种部署
│   ├── SDK 模式 / Gateway 模式 双签名
│   └── ✅ 本 Skill 覆盖范围
│
└── 方案 C：自建对接（不推荐）
    ├── 自己实现签名/验签/回调处理
    ├── 工作量大，容易出错
    └── 仅在有特殊需求时考虑
```

## 二、三种部署方式对比

| 维度 | HTTP 云函数 | 云托管（Cloud Run） | 本地开发 |
|------|:----------:|:------------------:|:-------:|
| **运维** | Serverless，免运维 | Docker，需管理容器 | 自行维护 |
| **冷启动** | 有（~1-3s） | 无（常驻容器） | 无 |
| **成本** | 按调用量计费 | 按容器时长计费 | 免费（开发用） |
| **回调域名** | 需开启 HTTP 访问服务 | 自动有公网域名 | 需 ngrok 穿透 |
| **适用场景** | 低频调用、快速原型 | 高频服务、生产环境 | 调试、开发阶段 |
| **推荐度** | ⭐⭐⭐ 推荐 | ⭐⭐⭐ 推荐（生产） | ⭐⭐ 仅调试 |
| **签名模式** | Gateway 更省心 | SDK 更灵活 | SDK（无集成中心） |

### 选择建议

```
你的场景是？
├── 初次接入 / 快速验证
│   └── → HTTP 云函数 + Gateway 模式（最快上手）
│
├── 生产环境 / 高并发
│   └── → 云托管 + SDK 模式（稳定可靠）
│
├── 已有 Docker 化项目
│   └── → 云托管（无缝迁移）
│
├── 排查问题 / 开发新功能
│   └── → 本地开发 + SDK 模式（快速迭代）
│
└── 不确定
    └── → 先用云函数跑通，后续迁云托管
```

## 三、四种支付方式对比

> 详细 API 层面对比见 [wechatpay-basic-payment](../wechatpay-basic-payment) skill。

| 维度 | JSAPI | H5 | Native（扫码） | APP |
|------|:-----:|:--:|:-------------:|:---:|
| **使用端** | 微信内（小程序/公众号） | 手机浏览器（非微信内） | PC 网页 | 移动 APP |
| **必传 openid** | 是 | 否 | 否 | 否 |
| **场景信息** | 可选 | **必填**（IP + h5_type） | IP 可选 | - |
| **授权目录** | 不需要 | **必须配置** | - | - |
| **AppID 类型** | 小程序/公众号 | 公众号 | 小程序或公众号 | 移动应用 |
| **调起方式** | `wx.requestPayment` | 跳转微信中间页 | 展示二维码 | `wx.requestPayment`(APP) |
| **结果确认** | 前端回调 | 前端回调 | **需轮询查单** | 前端回调 |
| **pay-common 支持** | ✅ `/wxpay_order` | ✅ `/wxpay_order_h5` | ✅ `/wxpay_order_native` | ✅ `/wxpay_order_app` |

### AppID 类型对应关系

| 支付方式 | AppID 类型 | 来源 |
|---------|-----------|------|
| JSAPI（小程序） | 小程序 AppID | 微信公众平台 |
| JSAPI（公众号 H5） | 公众号 AppID | 微信公众平台 |
| H5 | 公众号 AppID | 微信公众平台 |
| Native 扫码 | 公众号或小程序 AppID | 均可 |
| APP | 移动应用 AppID | 微信开放平台 |

> **注意**：AppID 必须已在[商户平台](https://pay.weixin.qq.com/)绑定关联。

## 四、SDK 模式 vs Gateway 模式对比

> 完整说明见 [sign-mode.md](../模板接入/sign-mode.md)。

| 维度 | SDK 模式 (`signMode=sdk`) | Gateway 模式 (`signMode=gateway`) |
|------|:------------------------:|:-------------------------------:|
| **主动请求签名** | SDK 自签直连微信（相同） | SDK 自签直连微信（相同） |
| **回调接收方** | 自己的服务器 | CloudBase 集成中心 |
| **回调验签解密** | 自己处理 | 集成中心代劳 |
| **回调 URL** | 自己的公网域名 | 集成中心固定域名 |
| **私钥安全性** | 在环境变量中 | 同上（主动请求仍需要） |
| **灵活度** | 高 | 中（依赖集成中心） |

### 选择决策树

```
你用什么部署？
├── HTTP 云函数 + CloudBase 集成中心
│   └── → Gateway 模式（回调由集成中心代理，省心）
│
├── 云托管（独立容器）
│   └── → SDK 模式（完全自主控制）
│
├── 自建服务器
│   └── → 只能用 SDK 模式（无集成中心）
│
└── 本地开发
    └── → SDK 模式 + ngrok 穿透
```

## 五、pay-common 能力矩阵

| 能力 | 状态 | 说明 |
|------|:----:|------|
| JSAPI 下单 | ✅ | 小程序/公众号内支付 |
| H5 下单 | ✅ | 含 Origin 白名单安全中间件 |
| Native 扫码下单 | ✅ | 返回 code_url |
| APP 下单 | ✅ | 独立签名格式（sign 非 paySign） |
| 订单查询（商户单号/微信单号） | ✅ | 双路查询 |
| 关闭订单 | ✅ | 自动检查状态防误关 |
| 申请退款 | ✅ | 支持部分退款 |
| 查询退款 | ✅ | 退款进度跟踪 |
| 商家转账（免密小额） | ✅ | 0.3-2000 元 |
| 支付回调处理 | ✅ | 先应答后处理 |
| 退款回调处理 | ✅ | 先应答后处理 |
| 转账回调处理 | ✅ | 先应答后处理 |
| JWT 鉴权（CloudBase Auth） | ✅ | 自动提取 openid |
| 小程序云调用 | ❌ | 预留，暂未支持 |
| 虚拟支付 | ❌ | 不在范围内 |
