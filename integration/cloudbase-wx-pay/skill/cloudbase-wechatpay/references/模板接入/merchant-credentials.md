# 商户凭证准备指南

> 详细介绍如何从微信支付商户平台获取接入所需的全部凭证。
> 无论使用集成中心还是 CLI 自部署，凭证准备步骤完全相同。

---

## 概述

接入微信支付需要 **7 项凭证**，全部从微信支付商户平台获取。操作前请确保：

- 已拥有微信支付商户号（10 位数字）
- 以**超级管理员**身份登录[商户平台](https://pay.weixin.qq.com/)
- 商户号已与目标小程序/公众号完成绑定（商户平台 → 产品中心 → AppID 账号管理）

---

## 1. APIv3 密钥

**路径**：账户中心 → API 安全 → 设置 APIv3 密钥

**操作步骤**：

1. 登录商户平台，进入「账户中心 → API 安全」
2. 找到「APIv3 密钥」区域，点击「设置」
3. 自定义 **32 位字母/数字组合**（必须恰好 32 字节）
4. 通过超管确认后完成设置

**重要说明**：

- 该密钥**仅在设置时显示一次**，需妥善保存
- APIv3 密钥用于回调的 AES-GCM 解密及部分 V3 API 响应的验签
- ⚠️ **未设置 APIv3 密钥 = 收不到任何回调通知**（支付结果、退款、转账全部丢失）
- 商户平台「交易中心 → 通知查询」将显示"通知发送失败"

---

## 2. 商户 API 证书（私钥）

**路径**：账户中心 → API 安全 → API 证书 → 申请/下载

**操作步骤**：

1. 进入「API 安全 → API 证书」
2. 点击「申请证书」（首次）或「下载证书」（已有）
3. 通过超管手机验证
4. 下载证书压缩包并解压

**下载后需保留的文件**：

| 文件 | 用途 | 说明 |
|------|------|------|
| `apiclient_key.pem` | 商户 API 私钥，用于下单请求签名 | **必须保留** |
| 证书序列号 | 40 位十六进制，形如 `36048761817F82958048330A6301E922AB28A04D` | 在页面上展示，需复制 |

> `apiclient_cert.p12` 与 `apiclient_cert.pem` 在本方案中**不使用**，可忽略。

**注意事项**：

- 私钥文件 `apiclient_key.pem` 内容以 `-----BEGIN PRIVATE KEY-----` 开头
- 在环境变量中填入时，换行须使用字面 `\n`（两个字符），而非真实换行符
- 首尾不能有多余空格
- 可使用 `scripts/check_pem_format.py` 脚本校验格式

---

## 3. 微信支付公钥

**路径**：账户中心 → API 安全 → 微信支付公钥管理 → 查看/下载公钥

**操作步骤**：

1. 进入「API 安全 → 微信支付公钥」
2. 如尚未启用，需先申请开通
3. 下载公钥文件并记录公钥 ID

**需保留的内容**：

| 项目 | 格式 | 说明 |
|------|------|------|
| `wxp_pub.pem` | PEM 文件，以 `-----BEGIN PUBLIC KEY-----` 开头 | 微信支付公钥文件内容 |
| 公钥 ID | 形如 `PUB_KEY_ID_0114...` | 页面上展示的 ID 字符串 |

---

## 4. 公钥与证书的区别说明

### 微信支付公钥 vs 微信支付平台证书

微信支付 V3 验签历史上存在两种方案：

| 方案 | 说明 | 推荐度 |
|------|------|--------|
| **微信支付公钥**（推荐） | 一次下载，长期有效 | ✅ 推荐 |
| 微信支付平台证书 | X.509 证书，会过期，需调接口轮换 | ⚠️ 不推荐 |

**选择逻辑**：
- 如在集成中心表单或环境变量中填写了微信支付公钥 → `pay-common` 使用公钥方案验签
- 未填写微信支付公钥 → 自动走平台证书自动下载模式

### 微信支付公钥 vs 商户 API 证书

| 凭证 | 作用方向 | 用途 |
|------|---------|------|
| **微信支付公钥** | 微信 → 商户 | 验证微信下发给商户的响应与回调（代表微信侧身份） |
| **商户 API 证书/私钥** | 商户 → 微信 | 对商户发起的请求进行签名（代表商户侧身份） |

> ⚠️ 两者作用相反，请勿混淆！填错会导致签名/验签始终失败。

---

## 5. 凭证清单

完成上述三节后，应已获得以下 **7 项值**：

| # | 名称 | 环境变量名 | 形态 | 取值出处 |
|---|------|-----------|------|---------|
| 1 | 小程序 AppID | `appId` | 形如 `wxc1546068399a59fc` | 微信公众平台 |
| 2 | 商户号 | `merchantId` | 10 位数字 | 商户平台首页 |
| 3 | APIv3 密钥 | `apiV3Key` | 32 字符字母数字 | 见第 1 节 |
| 4 | 证书序列号 | `merchantSerialNumber` | 40 位十六进制 | 见第 2 节 |
| 5 | 商户 API 私钥 | `privateKey` | `apiclient_key.pem` PEM 内容 | 见第 2 节 |
| 6 | 微信支付公钥 | `wxPayPublicKey` | PEM 内容 | 见第 3 节 |
| 7 | 公钥 ID | `wxPayPublicKeyId` | `PUB_KEY_ID_...` | 见第 3 节 |

## 6. 填入位置

| 部署方式 | 填写位置 |
|---------|---------|
| **集成中心**（推荐） | CloudBase 控制台 → 集成中心 → 微信支付 → 创建集成表单 |
| CLI 部署 | `cloudbaserc.json` → `envVariables` 字段 |
| 云托管 | CloudBase 控制台 → 云托管 → 服务配置 → 环境变量 |
| 本地调试 | 手动 `export` 或启动脚本注入 |

### 集成中心操作流程

1. 进入 [CloudBase 控制台](https://console.cloud.tencent.com/tcb) → 选择环境 → 集成中心
2. 找到「微信支付」→ 点击「创建集成」
3. 填写集成名称（自定义，例如 `miniapp-wxpay`）
4. 填入上述 7 项凭证
5. 点击「创建」

**创建成功后系统自动完成**：
- 按集成名称创建一个 HTTP 云函数（函数名形如 `<集成标识>-<随机串>`，统称 `pay-common`）
- 生成回调域名与具体回调 URL：

| 资源 | 说明 |
|------|------|
| 回调基础域名 | 形如 `https://<集成标识>.integration-callback.tcloudbase.com` |
| 支付回调 | `/wechatpay/order` |
| 退款回调 | `/wechatpay/refund` |
| 转账回调 | `/wechatpay/transfer`（当前版本暂不支持集成中心代处理） |

> 支付回调 URL 需同步填入微信支付商户平台的「支付结果通知 URL」（商户平台 → 产品中心 → 开发配置）。

## 7. 常见凭证问题

| # | 问题 | 原因 | 解决方案 |
|---|------|------|---------|
| 1 | PEM 私钥被拒绝解码 | 粘贴时缺失首尾行或换行被替换 | 确保包含完整 `-----BEGIN/END PRIVATE KEY-----`，中间换行用 `\n` |
| 2 | 证书序列号不匹配 | 复制时遗漏或多复制了字符 | 重新从商户平台复制完整的 40 位十六进制序列号 |
| 3 | APIv3 密钥长度错误 | 不足或超过 32 字节 | 重新在商户平台设置 |
| 4 | 公钥 ID 与公钥不匹配 | 混用了不同时期的公钥 | 确保公钥文件和公钥 ID 来自同一次操作 |
| 5 | NOT_ENOUGH / 平台证书过期 | 商户已切换到公钥模式 | 切换为公钥模式并填写公钥 ID，详见 [troubleshooting.md](../问题排查/troubleshooting.md) §3.7 |
